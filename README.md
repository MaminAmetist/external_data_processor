# Внешняя обработка «Перепродажа»

Внешняя обработка для платформы 1С:Предприятие 8.3 (управляемые формы).  
Предназначена для отбора документов по условиям, а также для удаления из полученного отбора документов по заданным условиям.

## Функционал

### 1. Команда «Заполнить нужное»

- Формирует таблицу документов, удовлетворяющих условию:
  - Дата начала действия **первого активного договора** в ведомости **равна или не равна**
  - **Минимальной или максимальной** дате начала действия всех договоров в этой ведомости
- Пользователь выбирает комбинацию условий через интерфейс:
  - `Сравнение дат`: «Равна» / «Не равна»
  - `Дата договора`: «Минимум» / «Максимум»
- Результат отображается в табличной части формы (колонка с ссылками на документы)
- Строки сортируются по **возрастанию даты** документа
- Строки с последним статусом «**Акцептован**» (по данным регистра `ИсторияСтатусовДокументов`) выделяются **светло-зелёным цветом**

### 2. Команда «Удалить лишнее»

- Удаляет из таблицы строки, у которых в табличной части документа «Ведомость» **чётное количество договоров**
- Выводит сообщение с количеством удалённых документов и их номерами (сортировка по возрастанию номера)

**Пример вывода:**
Удалено документов: 4 (000000016, 000000032, 000000256, 000001024)

## Дополнительные возможности

- Выбор типа сравнения (равно / не равно)
- Выбор экстремума (минимум / максимум)
- Программное условное оформление строк (цветовая индикация статуса «Акцептован»)

## Технические детали

| Параметр | Значение |
|----------|----------|
| Платформа | 1С:Предприятие 8.3.25 |
| Режим запуска | Толстый / Тонкий клиент |
| Тип обработки | Внешняя (файл `.epf`) |
| Формы | Управляемые |

## Зависимости (объекты метаданных в конфигурации)

| Объект | Назначение |
|--------|-------------|
| `Перечисление.СтатусыДокументов` | Статусы документа (Создан, Внутренний, Акцептован, Заблокирован) |
| `Справочник.Договоры` | Договоры с реквизитами: Наименование, Активен, НачалоДействия |
| `Документ.Ведомость` | Документ с табличной частью «Договоры» |
| `РегистрСведений.ИсторияСтатусовДокументов` | История изменения статусов документов |

## Установка и использование

1. Открыть 1С в режиме «Предприятие»
2. **Файл → Открыть** → выбрать файл `ВОEffectiveMobile.epf`
3. В открывшейся форме выбрать параметры сравнения
4. Нажать **«Заполнить нужное»** — таблица заполнится подходящими ведомостями
5. При необходимости нажать **«Удалить лишнее»**

## Комментарии к реализации

- Код модуля формы разделён на клиентские и серверные процедуры (`&НаКлиенте`, `&НаСервере`)
- Запросы к регистрам и документам оптимизированы, используют временные таблицы и соединения
- Условное оформление реализовано **программно**
- Вывод информации пользователю реализован через `СообщениеПользователю`
